# 记账 App - 技术规格(05)

## 技术选型
- **单文件纯前端**:一个 HTML 文件,里面装好 CSS + JavaScript。
- 不用框架、不用构建工具、不装任何环境。
- 双击就能在浏览器打开用。

## 数据怎么存
- 用浏览器**本地存储(localStorage)**:记的账存在你这台机器的浏览器里,刷新、关掉再开都还在。

## 文件结构
- `index.html` —— 就这一个文件,页面、样式、逻辑、数据读写全在里面。(用 `index.html` 这个名,拖到网上就是默认首页,直接能访问。)
- `docs/` —— 放前面几步的设计文档(需求/产品/线框/风格/本文件)。

## 关键取舍(大白话)
- **好处**:不用装任何东西、双击即用、能直接把文件发给别人。
- **代价**:数据只在你这台机器的这个浏览器里 —— 换手机、换电脑看不到,手动清了浏览器数据会丢。
- 你是自己一个人日常用(见 01 需求),这么选最省事、最快能用,完全够。要多人/跨设备,是"以后再说"的升级。

## 为什么这么选
匹配需求:一个人用、要快、要能立刻上手。单文件 + 本地存储是达成这些的最轻方式,零门槛。
